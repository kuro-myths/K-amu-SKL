## 🐛 Melaporkan Kerentanan

Jika Anda menemukan kerentanan keamanan di K-amu SKL, **JANGAN** membuat issue publik.

### Langkah Pelaporan

1. **Hubungi langsung**: Kirim laporan melalui kontak di profil GitHub [@kuro-myths](https://github.com/kuro-myths)
2. **Sertakan detail**:
    - Deskripsi kerentanan
    - Langkah reproduksi
    - Potensi dampak
    - Saran perbaikan (jika ada)
3. **Tunggu respon**: Kami akan merespon dalam **48 jam**
4. **Koordinasi**: Kami akan bekerja sama dengan Anda untuk perbaikan

### Timeline

| Tahap             | Waktu               |
| ----------------- | ------------------- |
| Acknowledgement   | 48 jam              |
| Investigasi awal  | 7 hari              |
| Perbaikan         | 14-30 hari          |
| Disclosure publik | Setelah fix dirilis |

## 🙏 Penghargaan

Pelapor kerentanan yang bertanggung jawab akan:

- Disebutkan di Security Advisory (jika diizinkan)
- Mendapatkan kredit di release notes
- Dihargai sebagai kontributor keamanan

## ⚠️ Responsible Disclosure

Kami meminta Anda untuk:

- ✅ Memberikan waktu yang wajar untuk perbaikan sebelum publikasi
- ✅ Tidak mengeksploitasi kerentanan
- ✅ Tidak mengakses data pengguna lain
- ❌ Tidak melakukan denial of service
- ❌ Tidak memodifikasi data pengguna lain
